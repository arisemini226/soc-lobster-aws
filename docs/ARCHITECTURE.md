# SOC 龍蝦系統 — 兩隻龍蝦溝通方式

## 🐉 兩隻龍蝦角色

| 龍蝦 | 名稱 | 位置 | 角色 |
|------|------|------|------|
| 協調者 | **小黑隊長** | Zeabur（這台）| 接收指令、協調任務、回覆結果 |
| 執行者 | **Hermes** | AWS Lightsail | 執行 AWS 命令、跑腳本、操作資安服務 |

## 📡 溝通流程

```
你（人類）
    ↓ Telegram / LINE
┌─────────────────────────┐
│ 小黑隊長                  │
│ • 接收指令                │
│ • 理解需求                │
│ • 規劃執行步驟            │
│ • 分配任務                │
└────────────┬────────────┘
             ↓
        我 SSH 進去
             ↓
┌─────────────────────────┐
│ Hermes（AWS Lightsail）  │
│ • 執行命令              │
│ • 跑腳本                │
│ • 串接 Bedrock AI       │
│ • 回傳結果              │
└────────────┬────────────┘
             ↓
        小黑隊長呈現
             ↓
┌─────────────────────────┐
│ 你（人類）               │
│ • 收到回覆              │
│ • 決定下一步            │
└─────────────────────────┘
```

## 🔧 我如何控制 Hermes

### 方式 1：SSH（主要方式）

```bash
ssh -i ~/.ssh/soc-hermes.pem ubuntu@<IP>
```

進入後可以：
- 執行 `openclaw` CLI
- 執行 `aws` CLI
- 跑 bash 腳本

### 方式 2：透過 Telegram

在 Telegram 對 Hermes 說：
- `資安狀態怎麼樣？`
- `Block IP 1.2.3.4`
- `建立測試 VPC`

Hermes 會自動執行並回覆你。

## 🎯 任務分配原則

| 任務類型 | 誰處理 | 原因 |
|----------|--------|------|
| 協調、規劃、回覆 | 小黑隊長 | 我在 Zeabur，反應快 |
| AWS CLI 操作 | Hermes | 在 AWS 環境，網路延遲低 |
| 資安腳本執行 | Hermes | 需要登入 AWS |
| 情資查詢 | 都可以 | 看緊急性質 |
| 緊急阻擋 | Hermes | 可以馬上執行 |

## 📝 溝通格式

### 你對我（小黑）說
任何格式都可以，我會理解後執行。

### 我對你回覆
- 直接 Telegram 回覆
- 複雜結果用 Markdown 格式化
- 需要行動時列出步驟

### Hermes 回覆
- Hermes 透過 Telegram 直接回覆你
- 或我轉達給你

## 🔄 備援模式

如果 Hermes 無法使用：

1. **小黑隊長直接操作**
   - 我用 AWS CLI 直接控制
   - 不經過 Hermes

2. **你手動操作 Lightsail**
   - 登入 AWS Console
   - 直接在網頁操作

## ⏱️ 回應時間

| 情境 | 預期回應時間 |
|------|-------------|
| 簡單指令（狀態查詢）| 3-5 秒 |
| 中等指令（建立資源）| 10-30 秒 |
| 複雜指令（Terraform apply）| 1-5 分鐘 |
| 緊急指令（阻擋 IP）| 5-10 秒 |
